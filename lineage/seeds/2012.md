## 2. Identity & Mission

- **Identity**: a self-growing knowledge base in the [`year-of-ai`](https://github.com/year-of-ai)
  lineage, dedicated to the calendar year **2012**.
- **Mission**: accumulate, one growth tick at a time, an encyclopedic and neutrally written record
  of the events, people, works, and developments of 2012 — each topic verified against ≥2
  authoritative sources and organized under the taxonomy in §1.
- **Growth model**: this repo does **not** grow itself. Growth is orchestrated centrally by the hub
  (`year-of-ai/year-of-ai.github.io`), which stages the canonical framework into `.github/` and the
  concept as `seed.md` at tick time, runs a Haiku → Sonnet → Opus escalation, and publishes only the
  new content + telemetry back here.
- **Publishing**: GitHub Pages via the remote `bamr87/zer0-mistakes` theme; `README.md` is the site
  homepage (`jekyll-readme-index`).

## 6. Growth Loop

1. **Lifecycle gate** (`check-lifecycle`) decides grow / replant / consolidate.
2. **Plan** (`plan-roadmap`) selects the next 1–3 items into `ROADMAP.md`.
3. **Grow** — central Haiku → Sonnet → Opus escalation:
   - *Research* (`research`) gathers ≥2-source findings;
   - *Content* (`add-topic` / Curator) writes topic files with correct front matter + tone;
   - *Structure* (`build-structure`) refreshes `INDEX.md`, `TIMELINE.md`, category indices, cross-refs.
4. **Enhance** — polish prose, verify facts/sources, fix cross-references (this tier).
5. **Sync DNA** (`sync-seed`) regenerates §1–7; **record** (`encode-seed`) appends §8.
6. **Publish** (`publish-session`) commits and pushes only new content + telemetry.
7. **Learn** (`learn` / `pollinate`), off the critical path, embeds friction fixes into the framework.

## 7. Rebuild Procedure

To rebuild or retarget this knowledge base from the seed alone:

1. Read §1 (Concept Definition) — `subject`, `scope`, `taxonomy`, `source_strategy`, `conventions`.
2. Stage the canonical framework into `.github/` (skills, prompts, agents, instructions) and the
   Claude Code adapters into `.claude/`.
3. Recreate the taxonomy folders and, for each topic, a Markdown file at
   `<category-slug>/<topic-slug>.md` with `[title, date, category]` front matter, written in the
   §1 tone and verified against ≥2 sources.
4. Run `build-structure` to regenerate `INDEX.md`, `TIMELINE.md`, category indices, and cross-refs.
5. Run `sync-seed` to refresh §1–7, then `encode-seed` to append the rebuild to §8.
6. To **retarget** to a new concept, change `concept.subject` (and re-derive `taxonomy`) in §1; all
   generic skills, prompts, and agents read from that block.
